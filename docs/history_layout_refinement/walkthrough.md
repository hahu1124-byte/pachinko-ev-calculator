# 履歴表示の不具合修正とレイアウト改善

履歴表示の利便性を向上させるため、詳細表示の不具合修正と簡略表示のレイアウト調整を行いました。

## 修正内容

### 1. 詳細履歴のチェックボックス修正

- `ui-manager.js` において、チェックボックスを `header-right` クラスで囲い、適切に配置されるように修正しました。
- `styles.css` に `header-left` と `header-right` のスタイルを追加し、機種名とチェックボックスが重ならないように整理しました。

### 2. 簡略履歴のレイアウト調整

- v55当時の表示に近い形式に戻すため、最上部に「日時（表示設定時のみ）」と「機種名」を配置するヘッダー行を追加しました。
- `compact-header`, `compact-date`, `compact-machine` クラスを導入し、視認性を高めました。
- チェックボックスの配置を `position: absolute` で右中央に固定し、操作性を改善しました。

## 検証結果

### 手動確認事項

- [x] 詳細表示で各アイテムの右側にチェックボックスが表示されていること。
- [x] 簡略表示で日時と機種名が1行目に表示され、その他のデータがその下に続くこと。
- [x] チェックボックスが正常に機能し、一括削除や統計への反映に問題がないこと。
